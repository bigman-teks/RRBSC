# Royal Rangers Bayelsa — Deployment Guide
## Complete Setup & Go-Live Instructions

---

## STEP 1: Create Firebase Project

1. Go to https://console.firebase.google.com
2. Click **"Add project"** → Name it `royal-rangers-bayelsa`
3. Enable Google Analytics (optional)
4. Click **Create project**

### Enable Services:
- **Authentication**: Build → Authentication → Get Started → Email/Password → Enable
- **Firestore**: Build → Firestore Database → Create database → Start in **production mode** → Region: `europe-west1` (or nearest)
- **Storage**: Build → Storage → Get Started → production mode
- **Hosting**: Build → Hosting → Get Started

### Get your config:
Project Settings (gear icon) → Your apps → Add app → Web → Register app  
Copy the `firebaseConfig` object.

---

## STEP 2: Paste Firebase Config

Open `assets/js/firebase-config.js` and replace:

```javascript
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",                    // ← paste your value
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};
```

Also update `APP_CONFIG`:
```javascript
const APP_CONFIG = {
  LOGO_URL: "https://yourlogourl.com/logo.png",   // ← paste logo URL
  PAYSTACK_PUBLIC_KEY: "pk_live_xxxxxxxxxxxx",     // ← your Paystack key
  // ... rest stays the same
};
```

---

## STEP 3: Set Firestore Security Rules

Firebase Console → Firestore → Rules → Replace with content of `firestore.rules` → Publish

---

## STEP 4: Set Storage Security Rules

Firebase Console → Storage → Rules → Replace with:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /profile_photos/{userId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
    match /events/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    match /news/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    match /gallery/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    match /manuals/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
    match /excos/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```

## STEP 5: Create Your First Super Admin

1. Go to your website → `register.html`
2. Register with YOUR email
3. Go to Firebase Console → Firestore → `members` collection
4. Find your document → click Edit
5. Change `role` field from `"member"` to `"super_admin"`
6. Change `status` field to `"active"`
7. Now login at `login.html` — you'll be redirected to `admin.html`

---

## STEP 6: Deploy to Firebase Hosting

### Install Firebase CLI:
```bash
npm install -g firebase-tools
```

### Login:
```bash
firebase login
```

### In your project folder:
```bash
firebase init hosting
```
When prompted:
- **Use existing project** → select `royal-rangers-bayelsa`
- **Public directory**: `.` (just a dot — since all HTML files are in root)
- **Single-page app**: No
- **GitHub auto-deploys**: Optional

### Deploy:
```bash
firebase deploy --only hosting
```

Your site will be live at:  
`https://royal-rangers-bayelsa.web.app`  
`https://royal-rangers-bayelsa.firebaseapp.com`

---

## STEP 7: Custom Domain (e.g. www.rrbayelsa.org)

Firebase Console → Hosting → Add custom domain → Enter domain  
Add the DNS records shown to your domain registrar (Namecheap, GoDaddy, etc.)  
SSL certificate is automatic — takes 24–48 hours to propagate.

## sw.js (Service Worker — Offline Support)

```javascript
const CACHE = "rr-bayelsa-v1";
const ASSETS = [
  "/", "/index.html", "/login.html", "/register.html",
  "/dashboard.html", "/excos.html", "/activities.html",
  "/manuals.html", "/admin.html",
  "/assets/css/main.css",
  "/assets/js/firebase-config.js",
  "/assets/js/auth.js"
];

self.addEventListener("install", e => {
  e.waitUntil(caches.open(CACHE).then(c => c.addAll(ASSETS)));
  self.skipWaiting();
});

self.addEventListener("activate", e => {
  e.waitUntil(caches.keys().then(keys =>
    Promise.all(keys.filter(k => k !== CACHE).map(k => caches.delete(k)))
  ));
  self.clients.claim();
});

self.addEventListener("fetch", e => {
  e.respondWith(
    caches.match(e.request).then(cached => cached || fetch(e.request))
  );
});
```

Register the service worker by adding this to the bottom of `index.html`:
```html
<script>
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js');
  }
</script>
```

## Admin Default Credentials Setup

After deploying:
1. Go to `yoursite.com/register.html`
2. Register the Super Admin account
3. Manually promote in Firebase Console (Step 5 above)
4. Login → Admin Panel → Settings → Add all other admins by email

**Super Admin can then:**
- Upload logo from Settings page
- Add all Excos profiles
- Create events and news
- Add other admin users by email
- Configure Paystack key

---

## Paystack Integration

1. Register at https://paystack.com
2. Go to Settings → API Keys & Webhooks
3. Copy your **Live Public Key** (starts with `pk_live_`)
4. Paste into Admin Panel → Settings → Paystack Public Key field
5. OR paste directly in `firebase-config.js` → `APP_CONFIG.PAYSTACK_PUBLIC_KEY`
